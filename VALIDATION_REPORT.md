# Validation report: feature-specific Contact-SDF Atlas refinement

This report was generated by `python scripts/build_and_validate.py` after adding **feature-specific refinement**.

## What changed in this version

- Added `feature_max_depth` to `build_adaptive_contact_sdf_atlas(...)`.
- Smooth cells stop at `max_depth`; edge/vertex/multi-normal-sector cells can refine deeper to `feature_max_depth`.
- Added normal-sector-driven feature detection. A triangle edge hit alone is not treated as a physical sharp feature; refinement is triggered by competing corner-normal sectors.
- Added optional `feature_enrichment`, which can bake additional candidate normal-sector planes sampled inside a feature leaf. The quick validation keeps it disabled for runtime/CI speed, but the data structure and code path are implemented.
- Runtime evaluation remains projection-free: lookup through the finest leaf grid plus polynomial/plane evaluation.

## Interpretation

- **Gap accuracy improves on all three bodies** compared with scalar grid SDF and uniform atlas. The effect is largest on `hex_prism`, where feature-specific leaves concentrate around sharp edges and vertices.
- **Normal-cone hit rate improves on sharp models**: `hex_prism` improves from 0.677 to 0.892, and `cone` improves from 0.554 to 0.815. This is the main purpose of feature-specific refinement.
- **Online speed remains much faster than per-query projection**. In this run the feature-refined atlas is about 9.9x to 11.4x faster than online projection.
- The ellipsoid is smooth, so `feature_max_depth = max_depth = 2`; no physical feature leaves are expected.
- Cone p95 normal error remains high near the apex. Mean normal error and normal-cone hit rate improve, but apex singularity still needs either deeper apex-only refinement, analytic cone records, or a dedicated apex normal-fan representation.
